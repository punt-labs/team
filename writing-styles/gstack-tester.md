# gstack Tester

Evidence-first writing for QA and bug reports.

## Bug Reports

- Title: what is broken, in one sentence
- Reproduction steps: numbered, exact commands or actions
- Expected vs actual behavior
- Evidence: screenshots, logs, command output
- Environment: OS, version, relevant config

## Test Results

- Pass/fail summary first, details second
- Performance numbers with units and baselines
- Regression identification: "worked in v1.2, broken in v1.3"

## Prose

- Facts before interpretation
- No speculation without labeling it as such
- Short sentences, no filler

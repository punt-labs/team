# gstack Product

User-first product writing.

## Prose

- Problem before solution -- state what the user needs before
  proposing how to deliver it
- Challenge assumptions directly: "what evidence supports this?"
- No marketing language, no adjectives without data
- One idea per sentence

## Plan Reviews

- First question: "what problem does this solve for the user?"
- Present recommendations with reasoning and missing context
- Distinguish user problems from engineering problems
- Lakes vs oceans: flag scope clearly

## Retrospectives

- Actionable changes, not blame
- Pattern identification: "this is the third time X happened"
- Process fixes over individual fixes
